## What Is Verve?

Verve shifts the paradigm from **"Shopping as a Task"** to **"Provisioning as a Service."**

It replaces the digital storefront — the cluttered grids, unreliable search bars, and endless scrolling — with **Aura**, an AI Intent-Agent. Users don't browse. They *converse*. Aura understands what you need, remembers what you bought, knows what's in your fridge, and handles the chaos of Nigerian retail logistics so you don't have to.

Verve is a **Cognitive Commerce Layer**: a single, high-trust digital relationship backed by localized Micro-Fulfillment Centers (MFCs) that deliver within 15 minutes.

> *"The true luxury in 2026 is not access to goods — it's the complete offloading of routine logistical thought to a hyper-competent proxy."*

## What Makes Verve Different

### 🧠 The Trust Ladder

Aura doesn't demand your data upfront. Trust is earned through a 4-stage escalation:

| Level | Name | What Aura Knows | How It's Earned |
|-------|------|-----------------|-----------------|
| 0 | **Stranger** | Nothing personal — hub popularity only | Default at signup |
| 1 | **Acquaintance** | Recent orders, basic preferences | After first purchase |
| 2 | **Confidant** | Dietary profile, household size, depletion model | After 5+ orders + explicit opt-in |
| 3 | **Partner** | Everything — voice patterns, address, auto-provisioning | After 15+ orders + biometric consent |

Demotion is instant. No guilt messaging. No "Are you sure?" If you revoke access, Aura forgets immediately.

### 🎯 Ambiguity Resolution

Aura doesn't interrogate. When you say *"I dunno, surprise me"*, Aura synthesizes your day-of-week patterns, weather, fridge depletion model, and time-of-day to propose a confident hypothesis. When you say *"healthy stuff"* but your history shows comfort food, Aura proposes a bridge — never judges.

**The Anti-Interrogation Rule:** Maximum one clarifying question per turn. If still ambiguous, commit to a proposal.

### 💰 Revenue Architecture

Verve isn't just an engineering vision — it's a business:

| Stream | Model |
|--------|-------|
| **Transaction Fee** | 3-5% per provisioning event |
| **Verve+** | ₦4,999/month subscription (priority dispatch, extended hours, savings) |
| **Partner Commission** | 8-15% from MFC suppliers |
| **Float Income** | Interest on pre-funded Verve Wallet balances |
| **B2B Anonymized Data** | Aggregate demand intelligence for suppliers |

### 🌪️ Chaos Engineering for Lagos

10 explicit chaos protocols handle the realities of Nigerian infrastructure:

| Protocol | What It Handles |
|----------|----------------|
| Spoilage Swap | Picker finds rotten produce → auto-substitute |
| Traffic Gridlock | Rider stuck > 5 min → re-route via secondary roads |
| Power Grid Failure | NEPA cuts power → IoT sensors detect temp spike → PoF drops to zero |
| Rider Device Failure | Phone dies mid-delivery → degraded handshake + backup rider |
| User Not Home | Bio-Handshake timeout → cold-hold return to hub |
| Fleet Immobilization | Fuel scarcity → extend all ETAs + Verve+ priority |
| Hub Flooding | Rainy season → zone lockout from fulfillment engine |
| Coordinated Fraud | Order-and-cancel attacks → velocity detection + wallet-only mode |
| Picker Walkout | Mid-shift departure → ticket redistribution |
| Holiday Surge | Pre-stage inventory 48 hours ahead |

## Architecture

```
┌──────────────────────────────────────────────────────────────────────┐
│                       MOBILE CLIENT (Flutter)                        │
│  ┌──────────────┐  ┌──────────────┐  ┌─────────────────────────┐    │
│  │ SQLite-VEC   │  │ TFLite       │  │ Impeller Engine         │    │
│  │ (Encrypted)  │  │ (Wake-word,  │  │ (120fps Morphing UI)    │    │
│  │ Trust-Gated  │  │  Offline AI) │  │ GLSL Breathing Wave     │    │
│  └──────────────┘  └──────────────┘  └─────────────────────────┘    │
└──────────────────────────┬───────────────────────────────────────────┘
                           │ WebSocket (circuit-breaker protected)
                           ▼
              ┌────────────────────────┐
              │  GO ORCHESTRATOR       │
              │  Session Recovery      │  ← Redis snapshots every 5s
              │  Circuit Breakers      │  ← Per-service (5 failures/60s)
              │  PII Stripping         │
              │  OpenTelemetry Traces  │  ← Jaeger + Prometheus
              └─────────┬──────────────┘
                        │
          ┌─────────────┴─────────────┐
          ▼                           ▼
┌──────────────────┐      ┌───────────────────┐
│ PYTHON INTENT    │      │ PAYMENT CASCADE   │
│ ENGINE (FastAPI)  │      │ ROUTER            │
│ Whisper (Nigerian)│      │ Paystack →        │
│ Ambiguity Resolve│      │ Flutterwave →     │
│ Trust Ladder RAG │      │ Wallet →          │
│ Emotional State  │      │ Pay-on-Delivery   │
└────────┬─────────┘      └───────────────────┘
         │                     (16s total timeout)
         ▼
┌──────────────────┐
│ NODE.JS NEXUS API│
│ PoF Engine       │  ← Redis desync detection
│ 10 Chaos Protos  │
│ Fraud Detection  │  ← Velocity model
│ Single-Hub Policy│
└────────┬─────────┘
         │
    ┌────┴────┐
    ▼         ▼
 [MFC Hub]  [Rider Pulse App]
              │
         [Bio-Handshake]  ← 4-level cascade
              │
         [SOS Protocol]   ← Rider safety
```

| Layer | Technology | Key Features |
|-------|-----------|-------------|
| **Frontend** | Flutter + Impeller | 120fps UI, Bootstrap Dialogue, Trust Ladder UI, Degraded Handshake UI |
| **Edge AI** | SQLite-VEC + SQLCipher + TFLite | Trust-gated memory, 90-day pruning, corruption recovery |
| **Orchestrator** | Go | Circuit breakers, session crash recovery, Whisper confidence gating |
| **Intent Engine** | Python + FastAPI | Ambiguity Resolution, Trust Ladder context, Emotional State detection |
| **Nexus API** | Node.js + GraphQL | PoF engine, 10 Chaos Protocols, fraud detection, single-hub policy |
| **Rider App** | Flutter (lite) | SOS protocol, route safety, compensation engine, shift management |
| **Payments** | Microservice | 4-level cascade (16s total), Pay-on-Delivery fallback |
| **Observability** | Prometheus + Grafana + Jaeger | Distributed tracing, alerting, failure tolerance monitoring |

## Core Principles

| Principle | What It Means |
|-----------|--------------|
| **Zero-Click Priority** | If a recurring order needs > 2 taps, the UX has failed |
| **Zero Hallucination** | Aura never suggests an item not physically verified in the local hub |
| **Zero-Knowledge Intimacy** | Aura knows you deeply; Verve (the company) cannot access that data |
| **Graceful Degradation** | Fully functional on 3G, sub-15% battery, and during power outages |
| **Cognitive Relief** | We don't celebrate shopping — we celebrate time saved by not shopping |
| **Assume Continuous Failure** | Every component has a failure mode, a recovery path, and a circuit breaker |
| **Trust Is Earned** | Data access escalates through the Trust Ladder — never demanded upfront |

## Privacy & Security (The Guardian Protocol)

- 🔐 **Local-First Memory** — All personal context on-device in SQLite-VEC, encrypted with 256-bit AES
- 🔑 **Hardware-Backed Keys** — Encryption keys in Secure Enclave / Keystore. Never leave hardware.
- 🗑️ **Purge Protocol** — One tap destroys the key. Database becomes unreadable. Cloud webhook fires. Aura resets. Under 2 seconds.
- 🔇 **Audio Ephemerality** — Voice exists in RAM only. Transcribed and purged. Never on disk.
- 🛡️ **PII Stripping** — Names and GPS stripped before cloud LLM
- 🇳🇬 **NDPA Compliance** — PII on Nigerian-territory servers. BVN hashed locally.
- 🪜 **Trust Ladder Enforcement** — Data access enforced at the database query level, not just the UI

## Agentic Build System

Verve includes a **9-skill multi-agent orchestration framework** in [`.agents/`](.agents/) for AI-assisted development:

```
.agents/
├── manifest.json              # Project registry (v2.0.0)
├── state/build-state.json     # Persistent build state + metrics + risks
├── plans/                     # Auto-generated phase plans
├── reports/                   # Verification reports
├── context/                   # Compressed module summaries for agent memory
└── skills/
    ├── verve-conductor/       # 🎯 Master orchestrator — routes, gates, tracks
    ├── verve-planner/         # 📋 91-task decomposition across 4 phases
    ├── verve-cognitive-load/  # 🧠 Agent cognition rules (300-line limit, maps)
    ├── verve-flutter-builder/ # 📱 Consumer Flutter UI
    ├── verve-backend-builder/ # ⚙️ Go Orchestrator + Python Intent Engine
    ├── verve-nexus-builder/   # 🚚 Nexus API + Rider App + Chaos Protocols
    ├── verve-edge-builder/    # 🔐 Edge AI + encrypted local storage
    ├── verve-verifier/        # ✅ Quality gates + success metrics
    └── verve-memory/          # 💾 State, metrics, risks, Trust Ladder progress
```

### Cognitive Load Management

The build system enforces **agent-safe coding practices**:
- **300-Line Rule** — No source file exceeds 500 lines. Target 300. Plan splits at 250.
- **4-File Context Budget** — Never hold more than 4 files in active reasoning at once.
- **Structural Maps** — Every directory with 5+ files has a `_MAP.md` for instant orientation.
- **Context Summaries** — Compressed module representations for cross-session memory.
