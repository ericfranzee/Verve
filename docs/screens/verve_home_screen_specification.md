# **Verve: Home Screen (The Cognitive Stage) UI/UX Specification v1.0**

**Document Status:** Final (Design/Engineering Ready) **Date:** May 8, 2026 **Target Architecture:** Flutter (Impeller Engine)

## **1\. Core Philosophy: The Zero-Click Canvas**

The Verve Home Screen is not a "page" you scroll through; it is a **responsive canvas** that morphs based on the intent processed by Aura. It eliminates the traditional tab bar, search bar, and product grid.  
The screen is vertically divided into two distinct interaction zones:

1. **The Morphing Viewport (Top 60%):** The dynamic visual output.  
2. **The Sentinel Zone (Bottom 40%):** The persistent auditory/haptic input interface.

## **2\. The Sentinel Zone (Bottom 40%)**

This area remains persistent during the entire "Provisioning Event." It is the anchor of the user's trust and interaction.

### **2.1 The Aura Breathing Wave (Central UI Element)**

* **Visual Appearance:** A horizontal, fluid waveform constructed using custom fragment shaders in Flutter.  
* **Colors & States (Internal Logic):**  
  * *Idle:* Deep Teal (\#008080). Slow, sine-wave oscillation. Represents readiness.  
  * *Listening:* Bright Cyan (\#22D3EE). Expands vertically based on microphone amplitude (PCM audio buffer).  
  * *Processing:* Amber (\#FFBF00). Fast, shallow ripples. Indicates the Go Orchestrator and LLM are assembling the intent.  
  * *Speaking:* Solid Teal. Waveform synchronizes precisely with the TTS (Text-to-Speech) audio output.  
* **External Function:** Tap the wave to interrupt Aura or initiate a new command. Long-press to activate "Offline Intent Buffer" if network is unstable.

### **2.2 Minimalist Ancillary Controls**

Situated discreetly below or around the Wave.

* **The "Vault" Icon (Bottom Left):** A small, subtle shield. Tapping it opens the "Guardian Protocol" drawer (Privacy, Settings, Memory Purge).  
* **The "Pulse" Icon (Bottom Right):** A subtle timeline icon. Tapping it pulls up the "Predictive Pantry" (draft baskets generated by decay modeling).  
* **Text Fallback Bar:** If the user is in a noisy environment (e.g., Lagos traffic), a single tap on a tiny keyboard icon converts the wave into a high-contrast text input field.

## **3\. The Morphing Viewport (Top 60%)**

This area dynamically changes states based on the UI-Metadata JSON payload sent by the backend. It uses Flutter's Stack and AnimatedSwitcher to ensure 120fps transitions.

### **State 1: "The Void" (App Launch)**

* **Looks:** Deep charcoal/true black gradient. Completely empty.  
* **Function:** Reduces cognitive load instantly. Focuses the user entirely on the Aura Wave.

### **State 2: "The Hero Card" (Product Mention)**

* **Trigger:** Aura says, *"I've staged the Salmon."*  
* **Looks:** A highly detailed, macro-photography image of the salmon slides in from the right. The image uses a **Parallax Tilt** (tied to device gyroscope) so the salmon appears to float slightly above the dark background.  
* **Information Overlay:**  
  * *Top Left:* Price (e.g., ₦12,500).  
  * *Bottom Left:* Verve Origin Tag (e.g., "Sourced: 6hrs ago, Lagos Hub").  
  * *Bottom Right (Emerald):* Vitality Score (e.g., "+High Protein").  
* **UX/Interaction:** Horizontal swipe to cycle through other items in the "Draft Basket." Vertical swipe down to reject/remove the item.

### **State 3: "The Proposal" (Bundles/Recipes)**

* **Trigger:** User asks for a "15-minute Keto dinner."  
* **Looks:** Three Hero Cards overlap like a premium deck of cards. The active ingredient (the one Aura is currently describing) smoothly scales up and moves to the front of the Z-index.  
* **The Guardian Shield:** A small shield icon on the card indicates Aura used "Intimate Context." Tapping it reveals: *"Suggested because you blocked carbs on Monday."*

### **State 4: "The GIS Canvas" (Fulfillment Tracking)**

* **Trigger:** User says, *"Confirm order."*  
* **Looks:** The product cards dissolve into particles (or slide down). The Viewport fades into a custom, dark-mode vector map.  
* **Elements:**  
  * *The Hub (Start):* A pulsing Verve icon.  
  * *The Rider (Moving):* A sharp, geometric chevron.  
  * *The Route:* A glowing Teal line.  
  * *Traffic Heatmap:* Subtle Amber/Red glowing zones indicating real-time gridlock (critical for Nigeria operations).  
* **Function:** Real-time location polling via the Nexus API.

### **State 5: "The Frosted Receipt"**

* **Trigger:** Simultaneous with State 4 (Map), or when user asks for a total.  
* **Looks:** A modal overlay using BackdropFilter (frosted glass blur effect) slides up over the lower part of the map.  
* **Elements:** Clean, monospaced typography detailing the transaction, VAT, and the final payment gateway used (e.g., "Paid via Paystack Token").

## **4\. The "Bio-Handshake" Overlay (Critical External UX)**

This is an absolute override state that occurs when physical fulfillment is imminent.

* **Trigger:** Rider GPS ping enters a 50-meter geofence of the user's location.  
* **Looks:** The entire screen (Viewport and Sentinel Zone) shifts to a single, vibrant, pulsing Hex color (e.g., Electric Blue). The Aura Wave is replaced by a large, centralized "Verve Pulse" graphic.  
* **External Function:** The user holds the phone up. The rider's app shows the exact same pulsing color (generated via a synchronized TOTP seed).  
* **Internal Function:** The Flutter app activates Bluetooth Low Energy (BLE) to exchange a cryptographic handshake with the rider's device.  
* **Resolution:** Screen flashes **Emerald Green**, haptic engine fires a "Double Heavy Pulse," and the UI returns to State 1 ("The Void") with a quiet *"Delivery complete."* from Aura.

## **5\. Technical Requirements for the UI**

To execute this UX flawlessly in the target environment:

### **5.1 Adaptive Fidelity Engine (Internal Logic)**

* **The Problem:** 4K Parallax images require heavy data and fast connections, which are not guaranteed.  
* **The Solution:** The app continuously pings the Go Orchestrator. If latency \> 400ms or connection type drops to 3G/Edge:  
  * The Viewport disables Parallax calculations.  
  * The UI rejects the .webp high-res image URLs from the backend payload.  
  * The app locally loads .svg silhouette vectors of the products (stored on-device) to ensure zero loading spinners.

### **5.2 Pre-Compiled Shaders**

* The "Breathing Wave" must not be drawn frame-by-frame using standard canvas commands (which causes jank on mid-tier Androids). It must be written as a custom fragment shader (GLSL) pre-compiled into the Flutter engine via Impeller.

### **5.3 The Audio-Metadata Sync Listener**

* The UI transitions (e.g., sliding the Hero Card in) must be tied to the audio playback head, not the network response time.  
* **Logic:** if (tts\_audio\_position\_ms \== ui\_metadata.timestamp\_ms) { triggerHeroAnimation(); }

## **Summary**

The Verve Home Screen is a reactive entity. It is dark, minimalist, and utilizes motion and sound to build trust. By replacing the cognitive burden of scrolling and searching with a highly synchronized Audio-Visual flow, Verve feels less like an application and more like a highly competent, invisible partner.
